# Plan 24-01: Sort Surveys by DateTime Descending

## Goal
Change survey sort order from ascending (oldest first) to descending (newest first) in `createFilteredSurveys`, so the SurveyPage and DashboardPage show the most recent surveys at the top.

## Current State
- `src/composable/useSurveyFilters.ts:97` — sorts by `date` ascending: `a.date.localeCompare(b.date)`
- This means oldest surveys appear first, forcing users to scroll down for recent ones

## Changes

### Task 1: Reverse sort order in `createFilteredSurveys`

**File:** `src/composable/useSurveyFilters.ts`

**Line 97** — Change:
```typescript
// Sort: oldest first (ascending order)
return filtered.sort((a, b) => a.date.localeCompare(b.date))
```
To:
```typescript
// Sort: newest first (descending order), then by time descending for same date
return filtered.sort((a, b) => b.date.localeCompare(a.date) || (b.time || '').localeCompare(a.time || ''))
```

**Rationale:**
- Reverses `a`/`b` in the comparator for descending date order
- Adds secondary sort by `time` descending so same-day surveys show latest time first
- Consistent with `createRecentFilteredSurveys` which already sorts newest first
- No other consumers of `createFilteredSurveys` need adjustment — SurveyComponent and DashboardComponent both benefit from newest-first

### Task 2: Update existing test to expect descending order

**File:** `src/composable/__tests__/useSurveyFilters.test.ts`

**Line 158** — Change test name from `'returns computed sorted by date ascending'` to `'returns computed sorted by date descending'`

**Line 168** — Change expected order from `['2', '1', '3']` (ascending: 08-01, 09-15, 11-20) to `['3', '1', '2']` (descending: 11-20, 09-15, 08-01)

## Verification
1. Surveys on SurveyPage display newest date first
2. Surveys on the same date sort by time (latest time first)
3. Dashboard filtered surveys also reflect newest-first order
4. No regressions in date filtering or search functionality
5. `yarn test:unit` passes with updated test assertion
