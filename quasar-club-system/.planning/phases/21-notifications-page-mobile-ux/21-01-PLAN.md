# Plan 21-01: Notifications Page Mobile UX

## Goal
Make the /notifications page fully usable on mobile by replacing the timeline layout with a card list on small screens and fixing cramped filter/action bars.

## Problems
1. `q-timeline` wastes ~50px horizontal space on mobile for timeline line/dots — content gets squeezed
2. `q-btn-toggle` with 4 filter buttons + `spread` is cramped on <400px screens
3. Actions bar row (filters + mark-all-read + refresh) wraps awkwardly into broken layout
4. Page has `q-pa-lg` padding — too much on mobile
5. Notification header row (title + chip + button) overflows horizontally

## Changes

### File: `src/pages/NotificationsPage.vue`

**Task 1: Conditionally replace q-timeline with simple card list on mobile**

Wrap the timeline in `v-if="!isMobile"` and add a mobile-specific card list with `v-else`. The mobile list uses `q-card` items directly (no timeline) to maximize horizontal space.

The `isMobile` ref is already imported from `useScreenComposable()` (line 215).

Mobile card list structure per notification:
```html
<q-card v-for="notification in filteredNotifications" flat bordered class="q-mb-sm notification-card">
  <q-card-section>
    <div class="row items-start no-wrap q-mb-xs">
      <q-icon :name="..." :color="..." size="sm" class="q-mr-sm q-mt-xs" />
      <div class="col">
        <div class="text-subtitle1 text-weight-medium text-black">{{ title }}</div>
        <div class="text-caption text-grey-6">{{ formatDateTime(...) }}</div>
      </div>
      <q-badge v-if="!notification.read" color="red" rounded floating />
    </div>
    <div class="text-body2 q-mt-sm">{{ message }}</div>
    <!-- invitation actions/status same as desktop but already fits -->
  </q-card-section>
</q-card>
```

**Task 2: Make filter bar scrollable on mobile**

Replace `q-btn-toggle` with individual `q-btn` chips in a horizontally scrollable container on mobile:

For mobile (inside `v-if="isMobile"`):
```html
<div class="filter-scroll" style="overflow-x: auto; white-space: nowrap; -webkit-overflow-scrolling: touch;">
  <q-btn
    v-for="opt in filterOptions"
    :key="opt.value"
    :color="filter === opt.value ? 'primary' : 'grey-3'"
    :text-color="filter === opt.value ? 'white' : 'grey-8'"
    :label="opt.label"
    rounded
    unelevated
    no-caps
    dense
    class="q-mr-sm"
    @click="filter = opt.value"
  />
</div>
```

For desktop, keep existing `q-btn-toggle`.

**Task 3: Stack actions bar vertically on mobile**

The existing `@media (max-width: 768px)` already sets `flex-direction: column` on `.actions-bar .row`, but the "Mark All Read" button with label is too wide. On mobile:
- Move the refresh button inline with filters
- Make "Mark All Read" full width below filters
- Use `icon-only` for mark-all-read on mobile (icon + tooltip, no label)

Change the actions bar to conditionally render:
- Mobile: filters row (scrollable) + mark-all-read icon button inline, refresh button inline
- Desktop: keep current layout

**Task 4: Reduce page padding on mobile**

Replace the static `q-pa-lg` class with conditional padding:
```html
<div :class="['notifications-page', isMobile ? 'q-pa-sm' : 'q-pa-lg']">
```

**Task 5: Make stats bar chips compact on mobile**

On mobile, use smaller chips without icons to save horizontal space. Replace the stats bar section:
- Mobile: single row with dense chips (no icon, just text + count)
- Desktop: keep current layout with icons

**Task 6: Update CSS media query**

Update the existing `@media (max-width: 768px)` block and add mobile-specific styles:

```css
@media (max-width: 600px) {
  .notifications-page {
    padding: 8px;
  }

  .notification-card {
    border-radius: 0;
    border-left: none;
    border-right: none;
  }

  .filter-scroll::-webkit-scrollbar {
    display: none;
  }
}
```
