# Plan 15-01: Auth Page i18n & Language Switcher

## Goal
Add i18n keys for auth pages, replace all hardcoded strings in login/register forms with `$t()` calls, add a language switcher to the AuthLayout, and fix the broken LanguageSwitcher.vue component.

## Requirements
- AUTH-01: Login page displays all labels, buttons, and links in the user's selected language
- AUTH-02: Register page displays all labels, buttons, and links in the user's selected language
- AUTH-03: A language switcher is visible on auth pages allowing CZ/EN switching before authenticating
- AUTH-04: Language selection persists to localStorage and carries over after login

## Tasks

### Task 1: Add auth i18n keys to both locale files

**Files:**
- `src/i18n/cs-CZ/index.ts`
- `src/i18n/en-US/index.ts`

**Changes:**
Add an `auth` section to both locale files with these keys:

```typescript
auth: {
  login: {
    title: "Přihlášení" / "Login",
    email: "E-mail" / "Email",
    password: "Heslo" / "Password",
    submit: "Přihlásit se" / "Log In",
    noAccount: "Nemáte účet? Zaregistrujte se" / "Not registered? Create an Account"
  },
  register: {
    title: "Registrace" / "Register",
    name: "Jméno" / "Name",
    email: "E-mail" / "Email",
    password: "Heslo" / "Password",
    submit: "Zaregistrovat se" / "Register",
    hasAccount: "Máte již účet? Přihlaste se" / "Already registered? Log in."
  }
}
```

### Task 2: Internationalize LoginFormNew.vue

**File:** `src/components/new/LoginFormNew.vue`

**Changes:**
- Import `useI18n` and destructure `t`
- Replace `<h3>Login</h3>` with `<h3>{{ $t('auth.login.title') }}</h3>`
- Replace `label="email"` with `:label="$t('auth.login.email')"`
- Replace `label="password"` with `:label="$t('auth.login.password')"`
- Replace `label="Login"` on q-btn with `:label="$t('auth.login.submit')"`
- Replace `Not registered? Create an Account` with `{{ $t('auth.login.noAccount') }}`

### Task 4: Fix LanguageSwitcher.vue component

**File:** `src/components/LanguageSwitcher.vue`

**Changes:**
Replace the entire component with a proper Quasar-styled toggle that:
- Uses `q-btn-toggle` with two options: `{ label: 'CZ', value: 'cs-CZ' }` and `{ label: 'EN', value: 'en-US' }`
- Reads initial locale from `useI18n()` locale
- On change: updates `locale.value`, saves to `localStorage.setItem('language', newValue)`
- Compact styling suitable for placement in the auth layout corner

### Task 5: Add language switcher to AuthLayout.vue

**File:** `src/layouts/AuthLayout.vue`

**Changes:**
- Import the fixed `LanguageSwitcher` component
- Add a `q-header` with a minimal `q-toolbar` containing the language switcher positioned to the right
- Keep the layout minimal — no navigation, just the language toggle in the top-right
- Use `flat` and `dense` styling so the header is unobtrusive

## Verification

After all tasks:
1. Navigate to `/login` — all text should be in the default language (Czech)
2. Click the language switcher to English — all labels, buttons, and links should update immediately
3. Navigate to `/register` — text should remain in the selected language
4. Refresh the page — language selection should persist (localStorage)
5. Log in — the app should use the same language selected on the auth page
