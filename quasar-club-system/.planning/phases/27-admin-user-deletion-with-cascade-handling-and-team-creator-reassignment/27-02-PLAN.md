  **useAdminUseCases.ts — Add deleteUserAsAdmin orchestration with audit logging:**
  - Import `useAuditLogFirebase` from the existing audit log service
  - Add `deleteUserAsAdmin` function accepting `uid: string` and `creatorResolutions: Array<{ teamId: string, action: 'delete' | 'reassign', newCreatorUid?: string }>`
  - Step 1: For each resolution with `action: 'delete'`, call existing `teamFirebase.deleteTeam(teamId)` — this handles full cascade (surveys, votes, messages, notifications, invitations, subcollections)
  - Step 2: Filter resolutions to only `action: 'reassign'` entries, pass them along with uid to `adminFirebase.deleteUserAccount({ uid, creatorResolutions: reassignResolutions })`
  - The Cloud Function handles: reassign creator fields, soft-delete user doc, delete Auth account
  - Step 3: After successful Cloud Function call, write audit logs (fire-and-forget pattern using `void writeAuditLog(...)`, same as existing codebase pattern in useCashboxUseCases.ts and useJoinRequestUseCases.ts):
    - For each reassign resolution: `void writeAuditLog({ teamId: resolution.teamId, operation: 'team.reassignCreator', actorUid: authStore.user.uid, actorDisplayName: authStore.user.displayName || authStore.user.email || 'Unknown', timestamp: new Date(), entityId: resolution.newCreatorUid, entityType: 'user', after: { previousCreator: uid, newCreator: resolution.newCreatorUid } })`
    - For the user deletion itself: `void writeAuditLog({ teamId: '', operation: 'user.delete', actorUid: authStore.user.uid, actorDisplayName: authStore.user.displayName || authStore.user.email || 'Unknown', timestamp: new Date(), entityId: uid, entityType: 'user', after: { deletedUserUid: uid } })` — use empty string for teamId since user deletion is cross-team. If writeAuditLog requires a teamId, use the first team the user belonged to, or create one log per team the user was a member of.
  - Add `getTeamsWhereUserIsCreator` helper: `(uid: string, teams: ITeam[]) => teams.filter(t => t.creator === uid)`
  - Export both new functions

  **useAdminComposable.ts — Add deletion UI state and logic:**
  - Add `showDeletedUsers` ref (boolean, default false)
  - Update `filteredUsers` computed: when `showDeletedUsers` is false, filter out users where `status === 'deleted'`. Apply this BEFORE the search query filter.
  - Add `deleteUser` async function: accepts `uid: string` and `creatorResolutions` array
    - Calls `adminUseCases.deleteUserAsAdmin(uid, creatorResolutions)`
    - On success: update `allUsers` ref — find user by uid and set `status: 'deleted'` and `deletedAt: new Date()` (local state update, no re-fetch needed)
    - Also update `allTeams` ref: remove teams that were chosen for deletion in resolutions
  - Add `getTeamsWhereUserIsCreator` wrapper that delegates to use case
  - Export: `showDeletedUsers`, `deleteUser`, `getTeamsWhereUserIsCreator`
