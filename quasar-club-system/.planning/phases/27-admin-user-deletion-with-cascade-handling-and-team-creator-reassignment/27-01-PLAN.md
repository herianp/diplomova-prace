<task type="auto">
  <name>Task 2: Create Cloud Function project and deleteUserAccount callable</name>
  <files>functions/src/deleteUser.ts, functions/src/index.ts, functions/package.json, functions/tsconfig.json, firebase.json</files>
  <action>
  1. Create `functions/` directory at project root (sibling to `src/`).

  2. Create `functions/package.json` with:
     - name: "functions"
     - scripts: `"build": "tsc"`, `"serve": "npm run build && firebase emulators:start --only functions"`, `"deploy": "firebase deploy --only functions"`
     - dependencies: `firebase-functions` (^6.x), `firebase-admin` (^13.x)
     - devDependencies: `typescript` (^5.x), `@types/node` (^20)
     - engines: `{ "node": "20" }`

  3. Create `functions/tsconfig.json`:
     - target: ES2022, module: commonjs, lib: ["ES2022"], outDir: "./lib"
     - strict: true, esModuleInterop: true, skipLibCheck: true
     - sourceMap: true, declaration: true
     - include: ["src/**/*"]

  4. Create `functions/src/deleteUser.ts`:
     - Import `onCall`, `HttpsError` from `firebase-functions/v2/https`
     - Import `initializeApp` from `firebase-admin/app`, `getAuth` from `firebase-admin/auth`, `getFirestore` from `firebase-admin/firestore`
     - Call `initializeApp()` at module level
     - Export `deleteUserAccount` as `onCall` with `{ region: 'europe-west1' }`
     - Verify caller has admin custom claim: `if (!request.auth?.token?.admin)` throw permission-denied
     - Accept `{ uid: string, creatorResolutions?: Array<{ teamId: string, action: 'delete' | 'reassign', newCreatorUid?: string }> }` as request data
     - Validate uid is provided, throw invalid-argument if missing
     - Step 1: Process creator resolutions if provided:
       - For `action: 'reassign'`: update team doc `creator` field to `newCreatorUid`, add `newCreatorUid` to `powerusers` array if not present (use `FieldValue.arrayUnion`)
       - For `action: 'delete'`: do NOT delete team here (client handles via existing `teamFirebase.deleteTeam()` BEFORE calling Cloud Function)
     - Step 2: Soft-delete user in Firestore: update `users/{uid}` with `{ status: 'deleted', deletedAt: new Date(), deletedBy: request.auth.uid }`
     - Step 3: Delete Firebase Auth account: `await getAuth().deleteUser(uid)`
     - Return `{ success: true }`
     - Wrap in try/catch: if Firestore soft-delete fails, throw internal error. If Auth delete fails after Firestore update, log warning but still return success (recoverable state per RESEARCH.md Pitfall 3).

  5. Create `functions/src/index.ts`:
     - Re-export: `export { deleteUserAccount } from './deleteUser'`

  6. Update `firebase.json`:
     - Add `"functions"` key: `{ "source": "functions", "runtime": "nodejs20", "predeploy": ["npm --prefix functions run build"] }`
     - Add `"functions"` to emulators: `{ "port": 5001 }`
     - Keep all existing config (firestore, hosting, storage, existing emulators) unchanged

  7. Run `cd functions && npm install` to install dependencies.

  IMPORTANT: The Cloud Function handles creator reassignment (updating team `creator` field) server-side using Admin SDK, which bypasses Firestore security rules. Team deletion for `action: 'delete'` is done client-side BEFORE calling the Cloud Function, using the existing `teamFirebase.deleteTeam()` cascade.
