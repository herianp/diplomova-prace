# Plan 22-01: Team Browse Scroll Container

## Goal
Wrap the team browse list in a scrollable container with a fixed max-height so it doesn't take over the full page in /teams and onboarding pages. Search input stays pinned above the scrollable area.

## Current State
`TeamBrowseList.vue` renders `q-input` (search) + `q-list` (teams) in a flat `<div>` with no height constraint. When many teams exist, the list grows unbounded, pushing other page content far down.

Used in 3 places:
- `TeamComponent.vue` (teams page) — inside a `q-card`
- `OnboardingWizard.vue` (desktop) — inside stepper step
- `OnboardingWizardMobile.vue` (mobile) — inside stepper step

## Changes

### File: `src/components/onboarding/TeamBrowseList.vue`

**Task 1: Wrap the q-list in a scrollable container**

Add a wrapper div with `max-height` and `overflow-y: auto` around the `q-list` element (line 36-88). The search input, loading state, and empty state banners remain outside the scroll container so they're always visible.

Change the teams list section from:
```html
<q-list v-else bordered separator rounded>
  ...
</q-list>
```

To:
```html
<div class="team-list-scroll">
  <q-list bordered separator rounded>
    ...
  </q-list>
</div>
```

**Task 2: Add scoped CSS for scroll container**

```css
<style scoped>
.team-list-scroll {
  max-height: 350px;
  overflow-y: auto;
}
</style>
```

350px fits ~6-7 team items (each ~48-52px), providing a compact view while showing enough results to be useful. Works well in both the onboarding wizard (which has a 450px min-height step) and the teams page card.
